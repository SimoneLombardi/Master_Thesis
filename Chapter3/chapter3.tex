\section{Problem description}
%The general task in all the performed test consists in controlling the end-effector \textit{position} and \textit{orientation} in a obstructed environment. The \textit{goal} has a known position and orientation, and the robot does not now a priori the position of the obstacles. The environment information are sent to the control algorithm as a set of points and vectors, representing the pair of point at minimum distance between the robot point cloud and the obstacle point cloud. These data is generated for each robot link. During movement it is supposed that contact with the robot constitutes a \textit{failure}, and also a time constraint is set for each task. The \textit{success} of each test is determined by the reaching of the target position with a cartesian error norm that is lower than $trsh = 0.03$. 
The primary task consist in controlling the robot's end-effector \textit{position} and \textit{orientation} within an obstructed environment. While the goal pose is known, the robot has no a priori knowledge of the obstacles' location. To navigate, the control algorithm receives environmental data as a set of \textbf{closest point pairs} and corresponding vectors, computed between each of the robot’s link point clouds and the obstacle point cloud. The sensor array is composed of time of flight sensors, and is simulated with a series of nodes that will be described in Chap.\ref{chap:fourth}. Any physical contact between the robot and the obstacles is categorized as a \textit{failure}, and each task must be completed within a specific time constraint. A test is deemed successful if the robot reaches the target with a Cartesian error norm below the threshold $trsh = 0.03$. 

\section{Task Priority}
As outlined in the Problem Description, the controller must simultaneously manage at least two concurrent objectives: a primary pose-reaching task defining objective and a secondary obstacle avoidance safety one. To resolve potential conflicts between these goals, a prioritized control scheme is employed. This approach utilizes null-space projection for lower-priority tasks, ensuring that the primary task does not interfere with the safety task. By decoupling these tasks, the robot maintains its trajectory toward the goal while reactively adjusting its configuration to ensure safety.
Continuing the discussion from \ref{equat:TaskPrioritiClassic_generalSolution}, the classic task priority algorithm described in \cite{slotine1991general} lacks the ability to smoothly activate and deactivate \textit{inequality} task when the robot is far from the boundary. This can lead to an \textit{over constraining} of the system. To avoid this, the implemented control algorithm is based on the work of \cite{simetti2016novel}, which incorporates in the IK equation an \textit{activation function} that is able to smoothly activate and deactivate inequality constraint tasks, such as the \textit{obstacle avoidance}.
The compact expression to compute the inverse kinematics at the \textit{k-th} priority level is similar to the classic approach:
\begin{equation}
	\begin{aligned}
		\vec{W}_k &= \vec{J}_k\vec{Q}_{k-1}(\vec{J}_k\vec{Q}_{k-1})^{\#,\vec{A}_k,\vec{Q}_{k-1}} \\
		\vec{Q}_k &= \vec{Q}_{k-1}(\mathbb{I}-(\vec{J}_k\vec{Q}_{k-1})^{\#,\vec{A}_k,\mathbb{I}}\vec{J}_k\vec{Q}_{k-1}) \\
		\vec{\rho}_k &= \vec{\rho}_{k-1} + \vec{Q}_{k-1}(\vec{J}_k\vec{Q}_{k-1})^{\#,\vec{A}_k,\mathbb{I}}\vec{W}_k(\vec{\dot{x}}-\vec{J}_k\vec{\rho}_{k-1})
	\end{aligned}
	\label{equat:Modern_TP_algorithm}
\end{equation}
\noindent The difference stands in the use of a new \textit{regularized pseudo-inversion operator} that integrates the \textit{activation function} as a weight matrix to modulate the solution of the algorithm to a specific task.\\
\noindent The operator is defined as:
\begin{equation}
	\vec{X}^{\#,\vec{A},\vec{Q}} \triangleq (\vec{X}^T\vec{A}\vec{X} + \eta(\mathbb{I}-\vec{Q})^T(\mathbb{I}-\vec{Q}) + \vec{V}^T\vec{P}\vec{V})^{\#} \vec{X}^T\vec{A}\vec{A}
	\label{equat:RegPseudoInverse_operator}
\end{equation}
where:
\begin{itemize}
	\item $\vec{A}$ : activation matrix for $k_{th}$ priority level
	\item $\vec{Q}$ : orthogonal projector for the null-space of task $k_{th}-1$
	\item $\vec{V}$ : right orthonormal matrix of the SVD decomposition for     $\vec{X}^T\vec{A}\vec{X}+\eta(\mathbb{I}-\vec{Q})^T(\mathbb{I}-\vec{Q})$.
	\item $\vec{P}$ : SVO regularization martrix, applied in \cite{nakamura1986inverse} 
\end{itemize}
\noindent And this approach also account for possible ill-conditioned pseudo inversion, using the aforementioned regularization operator.  


\subsection{Task Description}
This section will describe the mathematical formulation of the task I implemented in my control loop. To simplify the notation, all the matrices, if not stated otherwise, are projected on an inertial reference frame $\langle\textit{world}\rangle$.


\subsection{Obstacle Avoidance}
For the \textit{Obstacle Avoidance} task the approach proposed in \cite{maciejewski1985obstacle} was used. After computing the point of the robot body at \textit{minimum distance} from the obstacle, at said point the desired velocity was applied. In the opposite direction of the distance vector.
% single point
\\ \noindent \textbf{Task reference}:\\
\noindent The \textit{Reference rate} is considered as the \textit{desired shape} of the derivative of the variable to control.
To be more precise:
\begin{itemize}
	\item if $x > \bar{x}$ then $\dot{\bar{x}} < 0$
	\item if $x < \bar{x}$ then $\dot{\bar{x}} > 0$
	\item if $x = \bar{x}$ then $\dot{\bar{x}} = 0$
\end{itemize}
In the case of the \textit{obstacle avoidance} task, the controlled variable is the \textit{distance} of the robot body from the obstacles in the environment.
\noindent The \textit{distance} vector and the \textit{point} are retrieved in a topic, and are already projected in the $\langle\textit{world}\rangle$ reference frame.\\
\begin{equation}
	d = (P_o - P_r)\hspace{0.5cm};\hspace{0.5cm}\dot{\vec{\bar{x}}} = -d
\end{equation}
\begin{itemize}
	\item $P_o$ and $P_r$: point pair for \textit{Obstacle} and \textit{Robot}, that have the minimum distance from each other.
	\item $d$: minimum distance vector.
\end{itemize}
Since in the original paper the \textit{planar} and \textit{spatial} case are treated separately, in chap.\ref{chap:fifth} experiments will be conducted with both the vector as is, and without the $z$ component. To understand how the proposed approach behaves in the two cases. \\

\noindent \textbf{Activation function}: \\
Since this is a \textit{inequality} task, we want to avoid over-constraining the system. To do so, a smooth activation function was employed, whose value changes from $0$ to $1$ as the distance from the obstacle gets closer to the threshold.
The activation value, for a function that decreases as it gets further from a minimum value, is computed as follows:\\
\begin{equation}
	a = 
	\begin{cases}
		1 &  x < x_{min} \\[5pt]
		\frac{1}{2}[(\cos(\pi\frac{x-x_{min}}{\delta})+1)] & x\in[x_{min},x_{min}+\delta] \\[5pt]
		0 &  x > x_{min} + \delta
	\end{cases}
	\label{equat:activation_function_equation_decreasing}
\end{equation}
\noindent And for the opposite case, increasing going towards a maximum value:
\begin{equation}
	a = 
	\begin{cases}
		1 &  x < x_{min} \\[5pt]
		\frac{1}{2}[(\cos(\pi\frac{x-x_{min}}{\delta+\pi})+1)] & x\in[x_{min},x_{min}+\delta] \\[5pt]
		0 &  x > x_{min} + \delta
	\end{cases}
	\label{equat:activation_function_equation_increasing}
\end{equation}
\noindent Where:
\begin{itemize}
	\item $x$ : State variable to be controlled.
	\item $x_{min}$: Lowest value for the controlled variable.
	\item $x_{min}$: Highest value for the controlled variable.
	\item $\delta$: Interval length in which my \textit{activation value} should go from $0$ to $1$ or vice versa. 
\end{itemize}
\noindent Here a visualization for the profile of \ref{equat:activation_function_equation_decreasing} and \ref{equat:activation_function_equation_increasing} respectively:
\begin{figure}[H]
	\centering

	\includegraphics[scale=0.35]{ActFcn.eps}
	\caption{Generic decreasing and increasing activation function profile}
	\label{fig:DecreasingActFcn}
\end{figure}
\noindent
For the \textit{obstacle avoidance} task, the activation function is a diagonal $3\times 3$ matrix, which has a different activation value for each component of the \textit{distance} vector. The dimension for the planar case is $2\times 2$ considering only the $x$ and $y$ axis.

\noindent \textbf{Task Jacobian}:\\
For this task, different \textit{Jacobian} matrices must be computed depending on the link on which the \textit{minimum distance point} is located. To handle this efficiently, the Jacobians are constructed \textit{colum} by \textit{colum}, resulting in a more compact and readable implementation. The same method was used throughout the thesis.\\
\noindent 
In particular, the colum relative to the $i_{th}$ link is computed as:\\
\begin{equation}
	\leftindex^{w}{\vec{J}}_{i/w} =
	\begin{pmatrix}
		\leftindex^{w}{\vec{J}}_{i/w}^{L} \\[4pt]
		\leftindex^{w}{\vec{J}}_{i/w}^{A}
	\end{pmatrix} =
	\begin{pmatrix}
		\vec{R}^{w}_i \cdot {}^{i}\vec{ax}_{i} \times (r^{w}_n - r^{w}_i) \\[8pt]
		\vec{R}^{w}_i \cdot {}^{i}\vec{ax}_{i}
	\end{pmatrix}
	\label{equat:AnalyticJacobianFormula}
\end{equation}
\noindent where:
\begin{itemize}
	\item ${}^{w}\vec{J}_{i/w}$ : Jacobian matrix for the $i_{th}$ frame, with respect to the $\langle\textit{world}\rangle$ frame, projected in the $\langle\textit{world}\rangle$ frame.
	\item $\vec{R}^{w}_i$ : Rotation matrix, from the $i_{th}$ frame to $\langle\textit{world}\rangle$ the frame
	\item ${}^{i}\vec{ax}_{i}$ : axis of rotation for joint $i_{th}$ expressed in the joint frame.
	\item ${}^{w}\vec{r}^{i}_{n}=(r^{w}_n - r^{w}_i)$ : translation vector from $i_{th}$ frame to the \textit{end effector} frame, projected on $\langle\textit{world}\rangle$ frame.
\end{itemize}
\noindent The task jacobian of the \textit{obstacle avoidance} is computed up to the link containing the \textit{minimum distance point} or $mdp$. The following step computes the \textit{rigid body Jacobian} from the link $i_{th}$ to the $mdp$.
\begin{equation}
	\leftindex^{w}{\vec{J}}_{mdp/w} = 
	\begin{pmatrix}
		\leftindex^{w}{\vec{J}}_{mdp/w}^{L} \\[4pt]
		\leftindex^{w}{\vec{J}}_{mdp/w}^{A}
	\end{pmatrix} =
	\begin{pmatrix}
		\mathbb{I}_{3\times 3} & [r^{mdp}_{i}\times]^T \\[4pt]
		\vec{0}_{3\times 3} & \mathbb{I}_{3\times 3}
	\end{pmatrix} \cdot
	\begin{pmatrix}
		\leftindex^{w}{\vec{J}}_{i/w}^{L} \\[4pt]
		\leftindex^{w}{\vec{J}}_{i/w}^{A}
	\end{pmatrix} 
\end{equation}

\subsection{End Effector Target}
In this section the \textit{pose reaching} task for the end-effector is defined.\\
\noindent \textbf{Task reference}:\\
In this case, the task reference is the \textit{cartesian error} between the \textit{tool} frame and the \textit{end-effector} frame.
Computed in the simulation using the \textit{transformation buffer}, starting from names of the \textit{end effector} and \textit{tool} frame the linear error:
\begin{equation}
	err_{goal/tool}^{L} = \leftindex^{w}{(P_g - P_t)}
\end{equation}
all projected in the end effector frame. For the angular error, the quaternion expression of the two rotation matrices was used:
\begin{equation}
	\vec{R}_{goal}^{w} = \rho_{goal} \hspace{0,25cm};\hspace{0,25cm} \vec{R}_{tool}^{w} = \rho_{tool}
\end{equation}
I then compute the error using the quaternions that gives:
\begin{equation}
	\rho_{err} = \rho_{goal}^{-1} \cdot \rho_{tool}
\end{equation}
lastly this I took the vector part of this quaternion to multiply by the rotation $\vec{R}_{tool}^{w}$:
\begin{equation}
	\rho_{err} = 
	\begin{pmatrix}
		x_e \\
		y_e \\
		z_e \\
		w_e
	\end{pmatrix} =
	\begin{pmatrix}
		\vec{v}_e \\
		w_e
	\end{pmatrix}
	\hspace{0,25cm};\hspace{0,25cm}
	\vec{v}_e = \vec{u}\sin\left(\frac{\theta}{2}\right)
\end{equation}
and $\vec{v}_e$ is proportional to the angle error, for small angles. The vector I used as \textit{angular part} of the cartesian error is: 
\begin{equation}
	err_{goal/tool}^{A} = - \vec{R}_{tool}^{w} \vec{v}_e
\end{equation}

\noindent \textbf{Activation function}: \\
Since this is an \textit{equality task}, it has to be always active. And given the dimension of its \textit{reference rate}: $\dot{\bar{x}} \in \mathbb{R}^{6\times 1}$, the activation function is an identity matrix.
\begin{equation}
	\vec{A} = \mathbb{I} \in \mathbb{R}^{6\times 6}
\end{equation}
\noindent \textbf{Task Jacobian}:\\
The task jacobian is the \textit{complete geometric jacobian} for the \textit{end effector}, computed column by column as shown in \ref{equat:AnalyticJacobianFormula}.

\subsubsection{Goal broadcasting}
As the \textit{end effector} target for the \textit{reaching} task, a series of transformations was performed from the initial message that is sent through the \textit{Action server} as two vectors:
\begin{align}
	\vec{r} = \begin{pmatrix}
		x \\
		y \\
		z
	\end{pmatrix}
	\hspace{8pt} \text{;} \hspace{8pt}
	\vec{\rho} = \begin{pmatrix}
		\phi \\
		\theta \\
		\psi
	\end{pmatrix}
\end{align}

\noindent The first represents the desired translation, and the desired rotation as \textit{roll, pitch, yaw} angles.
The projection frames of this vector to define the goal position and orientation could be either the \textit{end effector} frame, or the \textit{world} frame. This was done for experimental purposes, for easily sequencing different goals, or to repeat reaching tasks to a specific position in the environment.

\noindent For the orientation of the goal, $\vec{\rho}$ is used to compose the rotation matrix that is then projected on the desired frame:
\begin{align}
	\vec{R}_{goal} &= \vec{R}_{z}(\psi) \cdot \vec{R}_{y}(\theta) \cdot \vec{R}_{x}(\phi) \\[4pt]
	\langle\textit{world}\rangle: \leftindex^{w}{\vec{R}}^{w}_{goal} &= \mathbb{I} \cdot \vec{R}_{goal} \\
	\langle\textit{end\_effector}\rangle: \leftindex^{w}{\vec{R}}^{ee}_{goal} &= \vec{R}_{ee}^{w} \cdot \vec{R}_{goal}
\end{align}
In the first case, the projection matrix is the identity since there is no intermediate frame.
The same process is done for the translation vector:
\begin{align}
	\langle\textit{world}\rangle: \leftindex^{w}{\vec{r}}^{w}_{goal} &= \vec{0} + \vec{r}_{goal}\\
	\langle\textit{end\_effector}\rangle: \leftindex^{w}{\vec{r}}^{ee}_{goal} &= \vec{r}_{ee}^{w} + \vec{r}_{goal}
\end{align}
Then the rotation matrix and the translation vector are use to publish the $\langle\textit{goal}\rangle$ in RViz.

\subsection{Joint limits}
This task is used to maintain the joint variables inside the \textit{mechanical limits} of the two robots, the values of those limits were retrieved by the respective data sheets.\\
\textbf{Task reference}:\\
\noindent Since the joint limit task has a range of acceptable value I want to maintain each variable inside this range.
The reference rate is computed as:\\
\begin{equation}
	\dot{\bar{q}}_{th} =  
	\begin{cases}
		\lambda \cdot (\lvert \overline{jl}_{th} - q_{th}  \rvert + \delta) &  q_{th} <= \overline{jl}_{th} \\[5pt]
		-\lambda \cdot (\lvert \overline{jl}_{th} - q_{th}  \rvert - \delta) &  q_{th} > \overline{jl}_{th} 
	\end{cases}
\end{equation}
Where:
\begin{itemize}
	\item $\overline{jl}_{th}$ is the average value of the upper and lower joint limit, for the $th$ joint
	\item $q_{th}$ is the position of the $th$ joint
\end{itemize}

\noindent \textbf{Activation function}:\\
The activation function value is the result of the sum of a decreasing and increasing \textit{bell-shaped function}, using the lower and upper joint limits as the extremes.\\
\noindent The resulting function has the following general shape:
\begin{figure}[H]
	\centering
	\hspace*{-3.0cm}
	\includegraphics[scale=0.4]{jl_activation_good.eps}
	\caption{Joint limits activation function profile, with generic values}
	\label{fig:jl_act_fnc}
\end{figure}
\noindent In the real task, the value for the $x$ is the $q$ of each joint, while the values of the activation function range are computed from the joint motion range. Where $x_{min}$ and $x_{max}$ are the range limits and $x_{avg}$ is the average value of the two.\\
\noindent \textbf{Task reference}:\\
Lastly, the task jacobian is an \textit{identity} $\mathbb{R}^{12\times 12}$ matrix, since each joint can be independently controlled.

\subsection{End Effector minimum altitude}
This task is used to keep the end effector away from the floor. This task was implemented to check the correctness of the \textit{task priority} algorithm, to have a more predictable task with which I could better assess the expected behavior of the system.\\
\\
\textbf{Task reference}:\\
\noindent 
The reference rate for this task is computed using the $z$ coordinate of the translation from the $\langle\textit{world}\rangle$ frame to $\langle\textit{end\_effector}\rangle$. 
\begin{equation}
	\textit{Reference rate: } \dot{\bar{x}}_z = \lambda \cdot (\vec{\bar{x}}_{min} + \delta - \vec{z}_{ee}^{w})
\end{equation}
The complete vector has all zero exept on the $z$ coordinate.
\begin{equation}
	\textit{Reference rate: } \vec{\dot{\bar{x}}} = [0, 0, \dot{\bar{x}}_z, 0, 0, 0]^T
\end{equation}
\noindent In the case of the \textit{Minimual altitude task} the variable I want to control is the $z$ of the \textit{end effector} of the robot, and to impose a minimum altitude $z_{min}$. Finally, the $\delta$ keeps count of the activation region, meaning that the task will be smoothly activated starting from $z_{min} + \delta$ to $z_{min}$.
\\ \\
\noindent \textbf{Activation function}: \\

\begin{figure}[H]
	\centering
	\hspace*{-3.0cm}
	\includegraphics[scale=0.4]{DecreasingActFcn_realValues_good_minalt.eps}
	\caption{Minimum altitude activation function profile with task-relevant values}
	\label{fig:DecreasingActFcn_rv}
\end{figure}
\noindent In this case, a $(6 \times 6)$ zero matrix was used as activation function, than the activation value was inserted in the third position of the major diagonal, ending up with:
\begin{equation}
	\vec{A}_{min alt} = \textit{diag} (0, 0, a_z, 0, 0, 0)
\end{equation}
using $z_min = 0,30$\textit{m} as my lower limit and $\delta = 0,1$\textit{m}, and the function is tuned to yield $y \in [0,1]$.
\\ \\
\noindent \textbf{Task Jacobian}:\\
The task jacobian is equal to the third row of the \textit{geometric jacobian} of the robot, to compute it the same method described in \ref*{equat:AnalyticJacobianFormula} was used, but with no rigid body trasformation.
